TECNOLOGÍAS DE WEB APIs

Comprendiendo las interfaces de programación de aplicaciones web

¿QUÉ ES UNA API?

Una API (interfaz de programación de aplicaciones) es un conjunto de reglas definidas que permiten que diferentes aplicaciones se comuniquen entre sí. Actúa como una capa intermedia que procesa las transferencias de datos entre sistemas.

Según Amazon Web Services (2023), las APIs son mecanismos que permiten a dos componentes de software comunicarse entre sí mediante un conjunto de definiciones y protocolos.

GET /api/users HTTP/1.1

Host: api.example.com

Authorization: Bearer token123

Accept: application/json

EJEMPLOS DE APIs POPULARES:

  • Las API de Google: Búsqueda, Gmail, Traductor o Maps
  • PayPal para procesamiento de pagos
  • API para incrustar videos de YouTube
  • WhatsApp Business API
  • APIs de redes sociales como Instagram y Facebook
FUNCIONAMIENTO DE UNA API

Las APIs funcionan como intermediarios entre diferentes sistemas, permitiendo que se comuniquen y compartan datos de manera eficiente y segura.

TECNOLOGÍAS FUNDAMENTALES DE WEB APIs

HTTP

HTTP (Hypertext Transfer Protocol) es el protocolo fundamental para la transferencia de datos en la web y la base de cualquier intercambio de datos en la World Wide Web. Es un protocolo de la capa de aplicación para sistemas de información distribuidos, colaborativos e hipermedia.

Características principales:
  • Sin estado (Stateless): Cada petición del cliente al servidor debe contener toda la información necesaria para entender la petición
  • Basado en el modelo cliente-servidor: El cliente inicia la comunicación enviando una petición, y el servidor responde
  • Métodos de petición: Define operaciones como GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD
  • Códigos de estado: Indican el resultado de la petición (200 OK, 404 Not Found, 500 Server Error, etc.)

Ejemplo de petición HTTP:

GET /api/products HTTP/1.1

Host: api.example.com

Accept: application/json

JWT (JSON Web Tokens)

JWT es un estándar abierto (RFC 7519) que define una forma compacta y autónoma para transmitir información de forma segura entre partes como un objeto JSON. Esta información puede ser verificada y confiable porque está firmada digitalmente.

Usos principales:
  • Autenticación: Una vez que el usuario inicia sesión, cada solicitud posterior incluirá el JWT
  • Autorización: Controlar a qué recursos puede acceder un usuario
  • Intercambio de información: Transmitir datos de forma segura entre partes

Un JWT consta de tres partes separadas por puntos: Header.Payload.Signature

Estructura de un JWT:

// Header: Algoritmo y tipo de token

{"alg": "HS256", "typ": "JWT"}

// Payload: Datos del usuario y metadatos

{"sub": "1234567890", "name": "John Doe", "iat": 1516239022}

// Signature: Garantiza la integridad

HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)

REST APIs

Las APIs REST (Representational State Transfer) son un estilo de arquitectura para diseñar aplicaciones en red. Utilizan métodos HTTP estándar como GET, POST, PUT y DELETE.

Principios REST:
  • Arquitectura cliente-servidor: Separación de responsabilidades
  • Sin estado (Stateless): Cada petición contiene toda la información necesaria
  • Cacheable: Las respuestas deben definir si son cacheables
  • Sistema en capas: El cliente no puede distinguir si está conectado al servidor final
  • Interfaz uniforme: Simplifica la arquitectura y mejora la visibilidad

SOAP APIs

SOAP (Simple Object Access Protocol) es un protocolo estándar de mensajería basado en XML para el intercambio de información estructurada en la implementación de servicios web. Proporciona un framework de mensajería que es independiente del sistema operativo, lenguaje de programación o plataforma.

Características principales:
  • Basado en XML: Formato estándar para todos los mensajes
  • Altamente estructurado: Reglas estrictas para la comunicación
  • WSDL (Web Services Description Language): Define formalmente la interfaz
  • Múltiples protocolos de transporte: Funciona sobre HTTP, SMTP, TCP, etc.
  • Seguridad integrada: WS-Security proporciona seguridad de extremo a extremo
  • Manejo de errores: Mecanismos estandarizados para reportar y manejar errores

GraphQL

GraphQL es un lenguaje de consulta para APIs y un tiempo de ejecución para cumplir esas consultas con los datos existentes, ofreciendo mayor flexibilidad que REST.

Ventajas de GraphQL:
  • Consultas precisas: El cliente especifica exactamente qué datos necesita
  • Una sola petición: Obtiene múltiples recursos en una sola solicitud
  • Tipado fuerte: Sistema de tipos que define claramente la API
  • Evolución de API: Añadir campos sin romper consultas existentes
  • Introspección: La API puede consultarse a sí misma para documentación

EJEMPLOS PRÁCTICOS

Ejemplo de petición REST con JavaScript 
// Petición GET a una API REST
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

// Petición POST a una API REST
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Ejemplo de uso de JWT
// Crear un JWT en Node.js
const jwt = require('jsonwebtoken');
const token = jwt.sign(
  { userId: 123, role: 'admin' },  // payload
  'secretKey',                     // clave secreta
  { expiresIn: '1h' }              // opciones
);

// Verificar un JWT
try {
  const decoded = jwt.verify(token, 'secretKey');
  console.log(decoded); // { userId: 123, role: 'admin', iat: 1516239022, exp: 1516242622 }
} catch(err) {
  console.error('Token inválido');
}
