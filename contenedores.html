CONTENEDORES

Tecnologías que revolucionan el despliegue de aplicaciones

¿QUÉ SON LOS CONTENEDORES?

Los contenedores son unidades ejecutables de software en las que se empaqueta el código de la aplicación junto con sus bibliotecas y dependencias, de manera que se puede ejecutar en cualquier entorno.

A diferencia de las máquinas virtuales, los contenedores virtualizan el sistema operativo en lugar del hardware, lo que los hace más ligeros y portátiles.

$ docker run -d -p 80:80 nginx

Este comando ejecuta un contenedor de Nginx en segundo plano, mapeando el puerto 80 del host al puerto 80 del contenedor.

BENEFICIOS DE LOS CONTENEDORES:

  • Portabilidad: funcionan igual en cualquier entorno
  • Eficiencia: utilizan menos recursos que las máquinas virtuales
  • Aislamiento: las aplicaciones no interfieren entre sí
  • Escalabilidad: fácil de escalar horizontal y verticalmente
  • Velocidad: se inician en segundos
MÁQUINAS VIRTUALES VS. CONTENEDORES
Máquinas Virtuales
  • Virtualizan hardware
  • Incluyen sistema operativo completo
  • Mayor consumo de recursos
  • Arranque más lento
  • Mayor aislamiento
Contenedores
  • Virtualizan el SO
  • Comparten el kernel del host
  • Menor consumo de recursos
  • Arranque en segundos
  • Aislamiento a nivel de proceso

DOCKER

Docker es una plataforma de código abierto que facilita la creación, implementación y ejecución de aplicaciones en contenedores. Proporciona un conjunto de herramientas que permiten a los desarrolladores empaquetar aplicaciones y sus dependencias en un contenedor virtual.

Componentes principales:
  • Docker Engine: El motor que permite crear y ejecutar contenedores
  • Docker Hub: Repositorio de imágenes de contenedores
  • Dockerfile: Archivo de configuración para crear imágenes
  • Docker Compose: Herramienta para definir y ejecutar aplicaciones multi-contenedor

KUBERNETES

Kubernetes (K8s) es una plataforma de código abierto para automatizar la implementación, el escalado y la gestión de aplicaciones en contenedores. Fue originalmente diseñado por Google y ahora es mantenido por la Cloud Native Computing Foundation.

Características principales:
  • Orquestación: Gestiona automáticamente los contenedores
  • Auto-escalado: Ajusta el número de contenedores según la carga
  • Auto-reparación: Reemplaza contenedores que fallan
  • Balanceo de carga: Distribuye el tráfico entre contenedores
  • Despliegue continuo: Facilita actualizaciones sin tiempo de inactividad

COMANDOS BÁSICOS DE DOCKER

Comando Descripción
docker pull [imagen] Descarga una imagen de Docker Hub
docker build -t [nombre] . Construye una imagen a partir de un Dockerfile
docker run [opciones] [imagen] Crea y ejecuta un contenedor a partir de una imagen
docker ps Lista los contenedores en ejecución
docker stop [contenedor] Detiene un contenedor en ejecución
docker rm [contenedor] Elimina un contenedor
docker images Lista las imágenes disponibles localmente
